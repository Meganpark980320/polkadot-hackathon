# ClubVault Pitch Deck

- 작성일: 2026-03-09
- 형식: 5장 발표용 초안

## Slide 1. Problem

제목:
`Student teams still manage money with chats, screenshots, and spreadsheets.`

핵심 포인트:
- 동아리, 스터디, 해커톤 팀은 공동 자금 관리가 수기 중심이다
- 누가 얼마나 냈는지, 누가 승인했는지 기록이 분산된다
- 팀 자금을 쓸 때 신뢰와 속도를 동시에 만족시키기 어렵다

발표 멘트:
`ClubVault는 이 혼란을 팀 단위 onchain treasury로 바꿉니다.`

## Slide 2. Solution

제목:
`ClubVault: shared treasury with proposal-based spending`

핵심 포인트:
- 누구나 금고 생성
- 멤버 공동 입금
- 지출은 proposal 생성 후 과반 승인 시에만 실행
- 승인과 집행 상태를 모두 UI에서 추적 가능

발표 멘트:
`학생 팀이 바로 이해하고 쓸 수 있는 가장 가벼운 공동지갑 경험에 집중했습니다.`

## Slide 3. Why Polkadot Hub

제목:
`Why Polkadot Hub EVM`

핵심 포인트:
- Solidity 기반으로 빠르게 개발 가능
- Polkadot Hub 테스트넷 대상 명확
- EVM 도구를 유지하면서 Polkadot 생태계 진입 가능
- 해커톤 제출 카테고리인 `DeFi / Stablecoin-enabled dapps`와 잘 맞는다

발표 멘트:
`ClubVault는 복잡한 DAO가 아니라, 실제 사용자 흐름을 Polkadot Hub 위에 얹는 제품입니다.`

## Slide 4. Demo

제목:
`End-to-end flow in under two minutes`

핵심 포인트:
- Vault 생성
- Deposit
- AI draft로 proposal 초안 생성
- Proposal 생성
- Approve
- Execute
- 마지막 tx hash 확인

발표 멘트:
`짧은 클릭 경로로 팀 treasury의 핵심 루프를 모두 보여줍니다.`

## Slide 5. Next Steps

제목:
`Next steps after hackathon`

핵심 포인트:
- Polkadot Hub TestNet 실배포
- explorer receipt UX 추가
- 팀별 preset / spending categories 추가
- 실제 학생 동아리 파일럿 테스트

발표 멘트:
`우리는 기능을 넓히기보다 실제 팀이 바로 써볼 수 있는 정도까지 제품을 다듬겠습니다.`
