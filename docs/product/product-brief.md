## 한줄 정의

`ClubVault`는 학생팀, 동아리, 스터디, 해커톤 팀을 위한 경량 공동지갑이다. 팀 자금을 개인 계좌에 맡기지 않고, 제안-승인-집행 흐름으로 투명하게 관리한다.

## 문제

소규모 팀의 돈 관리는 아직 대부분 수기다.

- 팀원들이 각자 송금하고 스크린샷을 공유한다
- 총무 한 명이 개인 계좌로 돈을 들고 있는 경우가 많다
- 어떤 비용이 왜 나갔는지 나중에 설명이 어려워진다
- "누가 승인했는가"가 명확하지 않다

이 문제는 단순한 가계부 문제가 아니라, `공동 자금에 대한 신뢰 비용` 문제다.

## 해결

팀이 공동지갑을 만들고, 지출은 아래 흐름으로 처리한다.

1. 팀 생성
2. 멤버 입금
3. 지출 제안 생성
4. 팀 승인
5. 자동 집행
6. 히스토리 조회

핵심은 돈이 특정 개인에게 귀속되지 않고, 집행 규칙이 코드로 고정된다는 점이다.

## 타깃 유저

- 대학 동아리 운영진
- 스터디 그룹
- 해커톤 팀
- 작은 프로젝트 팀

공통점:
- 팀 규모가 작다
- 돈은 자주 쓰지만 회계는 느슨하다
- 거버넌스 도구까지는 필요 없고, "가벼운 공동지갑"이 필요하다

## 가치 제안

- 개인 총무 없이 팀 자금을 관리할 수 있다
- 돈이 왜 빠졌는지 나중에 바로 설명할 수 있다
- 팀원 모두가 동일한 상태를 본다
- 멀티시그보다 쉽게 사용할 수 있다

## 왜 웹3인가

- 자금 보관 주체를 특정 개인에게 의존하지 않는다
- 승인 규칙이 온체인에 고정된다
- 집행 기록이 남는다
- 팀 내부 신뢰 비용을 낮춘다

즉, `ClubVault`는 회계 앱이 아니라 `shared treasury protocol with simple UX`에 가깝다.

## MVP 기능

- 팀 생성
- 멤버 등록
- 팀 자금 입금
- 지출 제안 생성
- 과반 승인
- 지출 집행
- 지출 히스토리 조회

이번 제출에서는 아래를 기본 규칙으로 잠근다.

- 제안 권한: 모든 멤버
- 승인 기준: 과반
- 집행 시점: 승인 즉시

## AI 보조 기능

AI는 제품의 중심이 아니라 보조 도구로 붙인다.

우선순위:
- 1순위: 자연어로 지출 제안 초안 생성
- 2순위: 이상 지출 플래그
- 3순위: 주간 자금 흐름 요약

원칙:
- AI가 없어도 제품이 완결되어야 한다
- AI는 프론트엔드/오프체인에서만 동작한다
- 최종 트랜잭션 승인 권한은 사용자에게 있다
